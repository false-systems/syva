# Syva v0.2 Release Checklist

Use this checklist before creating the v0.2 tag.

## Required Before Tag

Run the non-privileged gate:

```sh
make ci
```

Run the privileged release gates on a Linux host with BPF LSM enabled:

```sh
sudo -E make verify-runtime
sudo -E make verify-integration
sudo -E make verify-container-integration
sudo -E make verify-k8s-membership
sudo -E make verify-audit-mode
sudo -E make verify-network-lock
sudo -E make verify-cgroup-escape
```

Do not treat GitHub-hosted CI or Lima build checks as runtime enforcement
evidence unless the privileged commands above ran successfully there.

## Required Evidence

Capture the following in the release PR or release notes:

- `uname -a`
- `cat /sys/kernel/security/lsm`, including `bpf`
- release eBPF object path
- nine attached LSM hooks:
  - `syva_file_open` -> `file_open`
  - `syva_bprm_check` -> `bprm_check_security`
  - `syva_ptrace_check` -> `ptrace_access_check`
  - `syva_task_kill` -> `task_kill`
  - `syva_mmap_file` -> `mmap_file`
  - `syva_unix_connect` -> `unix_stream_connect`
- cgroup self-test passed
- inode self-test passed
- Unix stream self-test passed
- process/cgroup `file_open` block: `EPERM` and `deny_delta=1`
- container `file_open` block: `EPERM` and `deny_delta=1`
- Kubernetes annotated pod `file_open` block: `EPERM` and `deny_delta=1`
- Kubernetes membership evidence: pod name, node name, container ID, host
  cgroup path, cgroup inode/id, attach result, and detach result
- audit mode evidence: cross-zone read succeeded, would-deny decision counted
  (`deny_delta=1`), `enforcement_mode: "audit"` in `/healthz`
- network lock evidence: isolated-zone non-loopback connect/sendmsg/bind denied
  with `EPERM` (per-hook deny_delta=1), loopback allowed, and a bridged zone
  allowed out
- cgroup escape evidence: a zoned task moved out of its cgroup,
  `syva_cgroup_escape_detected_total=1`, security status degraded (detected,
  not prevented)

## Known v0.2 Gaps

Keep these explicit in release notes:

- Cgroup movement / zone escape is not *prevented* (`cgroup_attach_task` is not
  a BPF-LSM hook on supported mainline kernels). It is *detected* by the
  best-effort fentry detector (`syva_cgroup_escape_detected_total`), proven by
  `verify-cgroup-escape`.
- Kubernetes assignment is annotation-only; selector assignment, namespace
  defaults, leader election, finalizers, and CRD status remain out of scope.
- Only `file_open` has end-to-end process, container, and Kubernetes pod denial
  evidence.
- Runtime gates require a privileged BPF-LSM host.
- `INODE_ZONE_MAP` is keyed by inode only, not `(dev, ino)`.
- `/proc` and `/sys` coverage is incomplete.

## Guardrail Commands

Useful narrow checks:

```sh
make check-release-docs
make check-ebpf-artifact-policy
make proto-check
```

The release-doc drift check should fail on claims that are not the supported nine
hooks, claims that the removed cgroup-movement enforcement path is active,
missing release gate references, or claims that Lima alone proves runtime
enforcement without privileged BPF-LSM requirements.
